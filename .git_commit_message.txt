ci: add GitHub Actions workflow for documentation CI

Implement automated documentation quality checks using GitHub Actions
to validate MkDocs builds on push and pull requests to main and dev branches.


Features:
- Add .github/workflows/docs-ci.yml workflow for documentation CI
- Configure strict MkDocs build validation
- Add build artifact upload for successful builds
- Path-based triggering for docs-related changes only

Configuration:
- Update .readthedocs.yml to use pyproject.toml[docs] dependencies
- Use pip install -e ".[docs]" for dependency installation

Technical Details:
- Workflow triggers on: push/PR to main and dev branches
- Python version: 3.11 (matching production environment)
- Build mode: strict with verbose output
- Artifact retention: 7 days

Branch Configuration:
- Production branch: main → readthedocs.io/en/stable/
- Development branch: dev → readthedocs.io/en/latest/
