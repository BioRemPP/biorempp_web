Release v1.0.1 - Documentation CI Implementation

This release adds automated documentation quality checks and ReadTheDocs integration
for multi-version documentation deployment.

## What's New

### Features
- **GitHub Actions Workflow for Documentation CI** (#001)
  - Automated MkDocs build validation on push/PR to main and dev branches
  - Strict build mode with verbose error reporting
  - Build artifact upload with 7-day retention
  - Path-based triggering for efficiency (docs/, mkdocs.yml, pyproject.toml)

- **ReadTheDocs Multi-Version Support** (#002)
  - Configured automatic deployment for main (stable) and dev (latest) branches
  - Webhook integration for automatic builds
  - PDF and EPUB export formats enabled

### Configuration Updates
- Updated .readthedocs.yml to use pyproject.toml[docs] dependencies
- Standardized dependency installation: `pip install -e ".[docs]"`

### Branch Structure
- **main** → Production documentation (https://biorempp.readthedocs.io/en/stable/)
- **dev** → Development documentation (https://biorempp.readthedocs.io/en/latest/)

## Technical Details

**Workflow Configuration:**
- Python: 3.11 (matches production environment)
- Actions: checkout@v4, setup-python@v5, upload-artifact@v4
- Build: `mkdocs build --strict --verbose`

**ReadTheDocs Configuration:**
- OS: Ubuntu 22.04
- Build tool: Python 3.11
- Install method: pip with extra_requirements=[docs]
- Formats: HTML (default), PDF, EPUB

## Changes
- Documentation deployment now requires ReadTheDocs account and webhook configuration
- Local builds require installation with docs extras: `pip install -e ".[docs]"`

## Migration Guide
No migration required for existing deployments. New installations should:
1. Configure ReadTheDocs webhook in GitHub repository settings
2. Import project to ReadTheDocs
3. Configure versions: stable (main) and latest (dev)
